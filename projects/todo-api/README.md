# 📦 Todo API

Build a REST API to manage todos.

---

## 🎯 Goal

Learn how to design and build a basic backend service with CRUD operations.

---

## 🧩 Requirements

Your API should support:

- `GET /todos` → Get all todos  
- `GET /todos/:id` → Get a single todo  
- `POST /todos` → Create a todo  
- `PUT /todos/:id` → Update a todo  
- `DELETE /todos/:id` → Delete a todo  

---

## 🗂️ Data Structure

Each todo should have:

- id
- title
- completed (boolean)

---

## 💾 Storage

Start with:
- In-memory storage (array)

Optional:
- Upgrade to a database (SQLite, PostgreSQL, etc.)

---

## ⚠️ Error Handling

- Handle missing todos
- Validate input data

---

## 🚫 Constraints

- No full frameworks (keep it minimal)
- Focus on core logic

---

## ⭐ Bonus Challenges

- Add filtering (completed / not completed)
- Add pagination
- Add timestamps

---

## 📦 Submission

- PR with working API
- Include sample requests (curl/Postman)
