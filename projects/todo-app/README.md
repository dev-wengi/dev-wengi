# ✅ Todo App

Build a simple Todo application from scratch.

This is your first step into building interactive web apps — no tutorials, just you and the problem.

---

## 🎯 Goal

Learn the fundamentals of frontend development by building a working Todo app.

By the end, you should understand:
- How to handle user input
- How to update the UI dynamically
- How to manage simple application state

---

## 🧩 Requirements

Your app should allow users to:

- Add a new todo
- Mark a todo as completed
- Delete a todo
- View all todos

---

## 💾 Persistence (Optional → Recommended)

- Store todos in **localStorage**
- Data should persist after refresh

---

## 🎨 UI Expectations

- Clean and simple layout
- Each todo clearly visible
- Completed todos should look different (e.g. strikethrough)

---

## 🚫 Constraints

- No frameworks (React, Vue, etc.)
- No external libraries
- Keep it simple — focus on core logic

---

## ⭐ Bonus Challenges

- Edit a todo
- Add filters (All / Completed / Active)
- Add due dates
- Improve styling (make it look 🔥)

---

## 🧠 Hints (if you get stuck)

- You’ll need to listen to user events (click, input)
- Think about where your todos “live” (array?)
- After every change → update the UI
