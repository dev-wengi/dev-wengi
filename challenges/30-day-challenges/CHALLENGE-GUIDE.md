# 30 Nights of Building — Challenge Guide

## Purpose

This guide helps you turn the `30 Nights of Building` challenge into a daily practice. Use it to set up your workspace, stay consistent, and capture progress during the thirty-day journey.

## Setup Instructions

1. Open the `challenges/30-nights-of-building` folder in your editor.
2. Review the top-level `README.md` and choose the first project you want to build.
3. Open the target project folder and read its `README.md` for the expected outcome.
4. Create a dedicated branch or folder for daily progress if you want to track work separately.
5. Use version control commits each day to save progress and milestones.

## Daily Habit Tips

- Pick one project per day and focus on completing the core functionality first.
- Start each day by reviewing the project README and identifying the required features.
- Break the work into small chunks: design, implement, test, polish.
- Keep your code clean and your UI consistent.
- End each day with a short summary of what you built and what you want to improve next.

## Recommended Daily Workflow

- Morning: review the project requirements, plan the UI and data flow.
- Midday: build the core behavior and make the main feature work.
- Afternoon: add validation, responsive layout, and polish.
- Evening: test the app, fix bugs, and write a short progress note.

## Review Checklist

Use this checklist after finishing each project:

- [ ] Core workflow works end to end
- [ ] Required features are implemented
- [ ] UI is responsive on small and large screens
- [ ] Inputs are validated and errors are handled
- [ ] The project README explains the final expectations clearly
- [ ] The code is organized and easy to understand
- [ ] Project state persists when applicable
- [ ] The app is polished and ready for demonstration

## Stretch Goals

After completing the required features, try one or more stretch goals:

- Add a new improvement or feature for the project.
- Upgrade the UI with more refined styling.
- Add animations or transitions to improve the experience.
- Add search, filtering, or sorting for data-driven apps.
- Create a small demo video or GIF to showcase your finished project.

## How to Use the Guide

- Use `README.md` files for project requirements.
- Use this guide to stay consistent and avoid losing momentum.
- Return to the checklist each day to confirm the app is working well.
- Track progress in Git or a daily journal for accountability.
