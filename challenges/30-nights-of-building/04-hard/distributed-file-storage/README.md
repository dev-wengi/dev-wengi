# Distributed File Storage

Category: Hard

## End of 30-Day Goal

This Distributed File Storage should store files across multiple nodes with redundancy and accessible retrieval.

## Expected Features

- Upload and store files in a distributed manner.
- Retrieve files by key or path.
- Maintain redundancy or replication across nodes.
- Show storage health or node status.
- A clean interface for file management.

## Success Criteria

- Files are stored and retrieved reliably.
- Data distribution is handled clearly.
- The system demonstrates distributed file storage principles.
