# 30 Nights of Building

A thirty-day coding challenge with progressively harder project prompts. Each day focuses on one app or system design exercise, and the final result should be a polished, complete implementation of the selected challenge.

## Challenge Structure

The challenge is organized into five categories:

- `01-easy` — beginner-friendly UI-driven apps with a clear single workflow.
- `02-intermediate` — more advanced applications with data management, interactions, and productive user flows.
- `03-advanced` — AI-enabled, research-focused, or intelligent tooling experiences.
- `04-hard` — system-level engineering projects that emphasize architecture, distributed systems, and infrastructure.
- `05-nightmare` — ambitious platform-scale applications with multiple features and production-style workflows.

## Quick Start

1. Open `challenges/30-nights-of-building` in your editor.
2. Choose a project from the lists above.
3. Open the project folder and read its `README.md`.
4. Build the core features first, then polish the interface.
5. Use `CHALLENGE-GUIDE.md` for pacing, daily habits, and review checklists.

## How to Use This Challenge

1. Choose one project per day.
2. Read the project README in its folder for the expected final outcome.
3. Build the app or system with a polished UI, complete functionality, and a responsive experience.
4. Keep the code organized and document your work.

## Goal

Finish each project as a working, user-friendly implementation that demonstrates the core idea clearly and delivers a strong end-of-challenge result.
