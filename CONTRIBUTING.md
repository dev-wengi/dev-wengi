# Contributing to dev-wengi

Thanks for your interest in contributing!
We’re excited to have you here. Whether you're fixing a bug, adding a feature, or improving documentation, every contribution counts.

---

## 🛠 Getting Started

1. **Fork the repository**
   Create your own copy of the project on GitHub.

2. **Clone your fork**

   ```bash
   git clone https://github.com/your-username/dev-wengi.git
   cd dev-wengi
   ```

3. **Create a branch**
   Use a clear, descriptive name:

   ```bash
   git checkout -b feature/short-description
   ```

---

## Making Changes

* Keep changes focused and minimal (one feature/fix per PR).
* Follow existing code style and structure.
* Add or update documentation where necessary.

Commit your work with a meaningful message:

```bash
git commit -m "feat: add user authentication"
```

Push your branch:

```bash
git push origin feature/short-description
```

---

## Pull Request Process

1. Open a Pull Request against the `main` branch.
2. Clearly describe:

   * What you changed
   * Why it’s needed
   * Any relevant context or screenshots
3. Link related issues if applicable (e.g. `Closes #12`).

---

## 👀 Review Process

* Maintainers will review your PR and may request changes.
* Don’t take feedback personally — it’s about improving the project.
* Once approved, your contribution will be merged.

---

## Community Guidelines

* Be respectful and constructive.
* Assume good intent.
* Help others where you can.
* Read more here

---

## Reporting Issues

Found a bug or have a suggestion?

* Check if it already exists in the issue tracker.
* If not, open a new issue with clear details and steps to reproduce.

---

## Tips for Contributors

* Small PRs get merged faster.
* Clear descriptions save everyone time.
* When in doubt, ask!

---

Thanks again for contributing, you're helping make **dev-wengi** better for everyone.
